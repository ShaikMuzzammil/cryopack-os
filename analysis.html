📋 What is CryoPack OS?
CryoPack OS is a real-time vaccine cold-chain logistics optimizer. It solves the 3D bin-packing problem: given a set of vaccine boxes of varying sizes, pack them into the fewest possible cooling units (refrigerated transport containers) while maximizing space utilization. This directly reduces cold-chain costs, avoids vaccine spoilage, and ensures efficient vaccine distribution.
❄️
Why It's Needed
Vaccine cold-chain failures waste millions of doses. By minimizing the number of cooling units needed, CryoPack OS reduces refrigeration cost, transport weight, and the risk of temperature excursions. Fewer units = less cost + less waste.
📦
How Packing Works
Boxes are placed using the corner-point method: after each placement, 3 new candidate positions are generated at box corners: (x+w,y,z), (x,y+d,z), (x,y,z+h). Each algorithm scores these positions differently to achieve its unique packing behavior.
📊
Efficiency Score
Efficiency = total box volume ÷ (units used × unit volume) × 100%. The goal is to maximize this. A score of 85% means 85% of cold-chain space is being used for vaccines — only 15% is wasted air.
1. Input Boxes
User adds vaccine box dimensions (W×D×H) and quantities. Boxes can be added manually, randomly, or loaded from a saved dataset.
2. Sort by Algorithm
HYBRID & BFD sort by volume↓. FFD sorts by longest dimension↓. FF uses arrival order (no sort). This determines placement priority.
3. Corner-Point Placement
For each box, all candidate corner positions are evaluated. The best position is selected using the algorithm's scoring heuristic (surface contact, gravity, or sequential).
4. Unit Overflow
If no candidate position fits in the current cooling unit, a new unit is automatically opened. Boxes continue packing until all are placed.
5. Report & Compare
Efficiency %, cooling units used, placement log, and per-unit breakdown are shown. Compare all 4 algorithms to choose the best for your scenario.

⭐ RECOMMENDED
Hybrid Best-Fit
HYBRID — Surface-Contact Composite Scoring
Sort by volume↓. For each box, evaluate all open units using surface-contact scoring — counts how many faces of the placed box would touch walls, floor, or other boxes. Selects the unit with the highest contact score AND lowest remaining waste. Achieves the densest packing by minimising air gaps through physical surface maximisation.
65–88% efficiencySurface-contact scoreFewest unitsO(n²·m)
CLASSIC
Best Fit Decreasing
BFD — Gravity-Stack Heuristic
Sort by volume↓. For each box, finds the unit with the smallest remaining capacity that still fits — classic BFD rule. Uses a gravity-stacking placement: heavily penalises height (z), slightly penalises XY deviation from centre. Results in tight, low-centre-of-mass stacking distinct from Hybrid's surface-contact approach.
58–82% efficiencyGravity stackingVolume↓ sortO(n·log n+nm)
SPEED
First Fit Decreasing
FFD — Longest Dimension Sort
Unlike BFD, sorts by longest single dimension↓ (not volume) — this prioritises flat/elongated boxes differently. Places in the first unit with a valid corner-point using standard z→x→y ordering. No unit scoring performed — stops at first fit. Faster than BFD/Hybrid, different unit distribution due to sort order change.
50–75% efficiencyLongest-dim sortFirst-valid fitO(n·log n+nm)
ONLINE
First Fit (Online)
FF — Arrival-Order Spread Heuristic
No pre-sorting — boxes placed in arrival order. Uses a spread heuristic: sorts candidates by x+y first (spread across floor), then height. Distributes boxes widely before stacking tall. Worst-case approximation ≤17/10·OPT+2. Best for real-time streaming scenarios where boxes arrive sequentially.
